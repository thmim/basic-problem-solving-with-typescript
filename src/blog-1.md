any, unknown এবং Type Narrowing এর ধারনা

any the Type Safety Hole

TypeScript এর মেইন সুবিধা হচ্ছে কোড রান করার আগে এটি এরোর ধরতে পারে। তাই টাইপস্ক্রিপ্টে টাইপ উল্লেখ করে না দিলে এরোর দেখায়। তাই আমরা অনেক সময় অজনা টাইপ কে any দিয়ে ডিফাইন করি। এই ছোট একটা কারনই আমাদের কোডে বাগ ঢোকার জন্য যথেষ্ট। আজকের ব্লগে এটা নিয়েই আলোচনা করবো। টাইপস্ক্রপ্টে টাইপ any দেওয়ার  মানে হচ্ছে এই ভ্যারিয়েবলের মান যা আসবে তুমি নিষেধ করতে পারবা না। ধরেন আপনি আপনার বাসার দারোয়ানকে বললেন তুমি চোখ বন্ধ করে ঘুমায় থাকো বাসায় চোর আসুক আর বাসার লোক আসুক তুমি কাউকে আটকাতে পারবা না। তাহলে যে কাজটি হবে টাইপস্ক্রিপ্টে any ব্যবহার করলে সেই রকম কাজই হবে অর্থাৎ আপনার কোডে এরোর ঢুকে যাবে কিন্তু টাইপস্ক্রিপ্ট কিছুই বলবেনা। টাইপস্ক্রিপ্টের মুল কাজই হলো কোড রান করার আগে কোডের ভুল খুজে দেওয়া। কিন্তু যখনই আপনি কোন ভ্যারিয়েবলের টাইপ any দেন তখন টাইপস্ক্রিপ্ট ওই ভ্যারিয়েবলের উপর থেকে নজরদারি করা বাদ দেয়। যার ফলে কোডে একটি Hole তৈরি হয় যা দিয়ে খুব সহজেই বাগ ঢুকে যেতে পারে এবং এই কোড রানটাইমে অনেক সমস্যার সৃষ্টি করতে পারে। এজন্যই একে Type Safety Hole বলা হয়।

Example code:
let myData: any = 100;

টাইপস্ক্রিপ্ট এখানে কোনো এরর বা লাল দাগ দেখাবে না! 
কারণ 'any' ব্যবহারের কারণে সে টাইপ চেক করা বন্ধ করে দিয়েছে।
console.log(myData.length);

myData তে একটি নাম্বার রয়েছে। আমরা জানি নাম্বারের কখনো কোনো length থাকে না।তারপরও টাইপস্ক্রিপ্ট এখানে কোন এরোর দিবে না। তাই যখনই এই কোড রান করবেন তখনই আ্যপলিকেশনটি ক্রাশ করতে পারে।


unknown: The safer choice

যখন কোন ভ্যারিয়েবলের ডাটা টাইপ অজানা থাকে তখন any ব্যবহার না করে unknown ব্যবহার করা সবচেয়ে ভালো উপায়। unknown পুরোপুরি any  এর মতোই কাজ করে, এতেও আপনি যেকোন টাইপ ডাটা রাখতে পারবেন কিন্তু পার্থক্য হচ্ছে এরোর ধরার ক্ষেত্রে।টাইপস্ক্রিপ্ট টাইপ নিশ্চিত না হওয়া পর্যন্ত এই ভ্যারিয়েবলকে কোন কাজ করতে দিবে না।

Example code:

let myData: unknown = 100;

// টাইপস্ক্রিপ্ট এখানেই কোড আটকে দেবে এবং লাল দাগ (Error) দেখাবে!
console.log(myData.length);

এবার কিন্তু টাইপস্ক্রিপ্ট আপনাকে কোড রানই করতে দেবে না। সে বলবে আমি জানি না myData-র ভেতরে কী আছে। এটা তো string নাও হতে পারে তাই আগে কনফার্ম করো এটা কী, তারপর .length ব্যবহার করো।


Type Narrowing?

আগেই আমরা জেনেছি টাইপস্ক্রিপ্ট unknown ডেটা সরাসরি ব্যবহার করতে দেয় না। তাই একে ব্যবহার করার একটি দারুন উপায় হচ্ছে Type narrowing. type narrowing অর্থ হচ্ছে টাইপ সুনির্দিষ্ট করা।
একটি অজানা টাইপকে কন্ডিশন বা চেকিং এর মাধ্যমে একটি নির্দিষ্ট টাইপে আনার নামই type narrowing.

Example code:
function checkType (input: unknown){
    if(typeof input === "number" ){
        console.log('number')
    }
    else(typeof input === "string"){
        console.log('string')
    }

 }

 এক্সামপল কোড থেকে আমরা বুঝতে পারছি খুব সহজেই কিভাবে condition ব্যবহার করে type narrowing করে টাইপ কিভাবে নির্দষ্ট করা হয়।
